# FocusPet 反思与改进方案

> 文档版本：v3.1  
> 最后更新：2026-06-20  
> 当前状态：已从纯文档阶段进入可运行 Electron 原型阶段

---

## 1. 现状诚实评估

旧文档里写“代码零行、纯文档阶段”，这已经过时。当前项目已经具备：

- Electron 桌面窗口。
- 可拖拽桌宠。
- Canvas spritesheet 渲染。
- 任务管理。
- 番茄钟。
- 微启动。
- 应用/网页监测。
- 黑白名单与域名规则。
- 桌宠气泡拉回。
- 统计面板。
- 基础持久化。

当前真正的问题不再是“能不能跑起来”，而是：

1. 专注模式需要更聪明地理解上下文。
2. 桌宠交互需要更轻，不占屏幕。
3. 统计要从“数字面板”变成“能指导下一步”。
4. AI 功能要避免变成花活，优先服务任务启动和拆解。

---

## 2. 已经验证正确的方向

### 2.1 低压力拉回优于硬拦截

对 ADHD 用户来说，强拦截容易带来挫败感和反抗心理。当前的桌宠气泡、emoji 快捷按钮、微启动，比全屏锁更适合作为默认体验。

硬拦截可以作为可选严格模式，不应成为默认。

### 2.2 微启动命中度很高

“试 2 分钟 / 试 5 分钟”比“开始 25 分钟”更容易让用户动起来。当前已经实现，应该继续围绕它做：

- 续杯。
- 完成鼓励。
- 小步任务拆解。
- 当前任务锚点。

### 2.3 白名单不能只看应用名

浏览器里可能是：

- GitHub
- 文档
- Kimi
- Claude
- 豆包
- localhost
- 短视频

只看 Chrome/Edge 没意义。当前已开始按 URL/域名/标题综合分类，但可靠性仍需优化。

## 3. 当前实现与旧 PRD 的差异

| 旧 PRD 设想 | 当前实际策略 |
|---|---|
| 应用锁强拦截 | 先做轻量提醒与评分 |
| 认知校验弹窗 | 仅作为实验组件保留，长期迁移为桌宠气泡 |
| AI 伴侣对话优先 | 先做任务拆解与快捷动作 |
| 多桌宠/装扮 | 暂缓 |
| 养成商店 | 暂缓 |
| 面板大而完整 | 逐步转向轻唤醒 + 小占位 |

## 4. 主要风险

### 4.1 URL 获取不稳定

Windows 读取浏览器地址栏依赖 UIAutomation，可能受浏览器版本、语言、焦点状态影响。macOS 读取 URL 需要系统自动化权限。

缓解：

- 获取 URL 成功时按域名判断。
- 失败时按标题关键词判断。
- 允许用户手动补充域名规则。
- 后续考虑浏览器扩展辅助。

### 4.2 黑白名单配置成本高

如果用户要手动维护大量规则，产品会失败。

缓解：

- 默认内置常见 AI、开发、文档、学习域名。
- 专注模板：编程 / 写作 / 学习 / 严格。
- 后续做“从当前应用/网站一键加入白名单”。

### 4.3 桌宠可能分散注意力

桌宠如果过度动，会从“注意力锚点”变成“干扰源”。

当前策略：

- 专注中保持稳定。
- 待机低频随机动作。
- 提醒用气泡，不用大弹窗。

后续建议：

- 增加安静模式。
- 增加专注中动画强度设置。
- 把随机认知校验从面板弹窗迁移到桌宠气泡，保持提醒入口一致。

### 4.4 计时器生命周期问题

已修复：计时器从 `TimerPanel` 移到 `App` 全局层，切换页面/收起面板后仍继续计时。

后续仍建议：

- 进一步移到主进程，避免渲染进程挂起时误差。
- 记录实际开始时间和结束时间，用 wall clock 校正。

## 5. 下一阶段路线

### 阶段 A：让专注模式真的聪明

- 任务绑定期望应用/域名。
- 当前任务与当前应用上下文匹配。
- Kimi/Claude/豆包/ChatGPT 网页默认更智能放行。
- 浏览器 URL 识别增强。

### 阶段 B：让任务更容易启动

- `✨ AI 拆解` 接 LLM。
- 大任务自动拆成 5-8 个可执行小步。
- 支持“只做下一步”模式。
- 支持任务卡点记录。

### 阶段 C：让统计有指导意义

- 今日/7天趋势图。
- 最容易分心域名。
- 最容易恢复的提醒方式。
- 最适合专注的时间段。

### 阶段 D：轻量化桌宠入口

- hover 小按钮打开面板。
- 托盘重置位置。
- 面板只在需要时展开。
- 收起态进一步减少遮挡。

### 阶段 E：减少测试与工程摩擦

- 收敛 `index.ts` / `index.cjs` 双入口维护。
- 增加日志文件和“复制诊断信息”能力。
- 给 URL 监测失败提供可见原因，例如缺少 macOS 自动化权限。
- 用 wall-clock 或主进程计时器修正长时间后台运行误差。

---

## 6. 当前优先级结论

短期最值得继续做的是：

1. 任务上下文匹配。
2. AI 任务拆解接 LLM。
3. URL 识别稳定性。
4. 统计趋势图。
5. 面板 hover 唤醒。

暂时不要投入大量时间在：

- 多桌宠资产。
- 装扮系统。
- 商店/成就。
- 复杂全屏锁。

一句话总结：现在 FocusPet 已经不是“先让精灵动起来”的阶段了，而是要让它真正知道“用户此刻应该回到哪里”。

## 1. 产品定位

FocusPet 是一款桌面级专注辅助工具。它不是普通番茄钟，也不是单纯的屏蔽器，而是一只常驻桌面的“注意力伙伴”：通过桌宠状态、气泡提醒、任务上下文、应用/网页监测和微启动机制，帮助用户从分心中温和回到工作。

当前产品尤其关注 ADHD 友好的体验：

- 降低启动成本：2/5 分钟微启动，而不是一上来要求 25 分钟。
- 外部化当前任务：桌宠提醒“现在要回到什么”。
- 低羞耻拉回：不弹大警告，不骂用户，用气泡和快捷按钮轻推。
- 减少配置负担：专注模板、默认 AI/开发/文档白名单。
- 允许合理漂移：Kimi、Claude、豆包、ChatGPT 等 AI 网页在多数专注场景下默认放行。

## 2. 当前已实现能力

### 2.1 桌宠窗口与交互

- Electron 无边框透明悬浮窗。
- 桌宠可拖动到屏幕任意位置。
- 窗口位置持久化。
- 收起态占位缩小为桌宠本体附近区域。
- 面板展开方向会根据桌宠位置向左/向右适配。
- 右键/双击桌宠打开面板。
- 托盘支持显示、打开面板、开始专注、置顶、穿透、开机自启、退出。
- 待机状态低频随机小动作，专注中保持安静。

### 2.2 桌宠渲染与状态

- Canvas 2D spritesheet 渲染。
- 兼容 8×9、192×208 的 pet atlas 格式。
- 当前内置像素猪猪侠桌宠资源。
- 状态机支持：
  - idle
  - focus
  - celebrate
  - sad
  - angry
  - sleep
  - alert
  - drag / air
- 粒子特效：星尘、火花、雨滴、Zzz 等。

### 2.3 任务系统

- 添加 / 删除 / 完成任务。
- 任务优先级：高 / 中 / 低。
- 任务支持：
  - 有时间段任务
  - 无时间段任务
  - 提醒时间
  - 预计番茄数
- 任务视图：
  - 日程
  - 待排
  - 全部
  - 完成
- 批量导入任务。
- 从剪贴板一键导入。
- 任务到点/逾期时由桌宠气泡提醒。
- 任务旁 `✨` 拆解按钮：当前为本地模板拆解，会追加 5 个子任务。

### 2.4 专注模式

当前专注模式是“低压力注意力护栏”，不是强制锁死电脑。

开启后会：

- 后台计时，即使切到任务页/统计页/收起面板也继续走。
- 自动绑定当前时间段或优先级最高的待办任务。
- 桌宠进入 focus 状态。
- 监测当前应用/网页上下文。
- 白名单应用/域名：认为仍在工作。
- 黑名单应用/域名：记录走神、扣评分、桌宠拉回。
- 中立应用：观察一段时间后温和提醒。
- 完成后记录专注历史、奖励星尘、触发鼓励气泡。
- 统计页可以选择专注模板：编程 / 写作 / 学习 / 严格。

已实现计时能力：

- 经典专注时长调节。
- 短休 / 长休调节。
- 4 轮后长休。
- 微启动：试 2 分钟 / 试 5 分钟。
- 微启动完成后桌宠询问“续杯吗”。
- 随机认知校验仍保留为实验性面板，后续应改成桌宠气泡式确认，减少打断感。

### 2.5 ADHD 友好拉回

已实现：

- 主要拉回动作通过桌宠主动气泡完成，不使用系统级弹窗。
- 气泡已收敛为低打扰纯提示，不再显示快捷按钮。
- 分级拉回：
  - 轻提醒
  - 任务锚点提醒
  - 10 秒落地动作
  - 建议短休
- 专注完成后鼓励用户：
  - 走一走
  - 喝水
  - 再来一轮
- 午饭 / 晚饭提醒。

随机认知校验已迁移为桌宠气泡提示，不再使用独立弹窗组件。

### 2.6 应用与网页监测

当前实现为轻量监测，不是硬拦截：

- Windows：PowerShell + user32.dll 获取前台应用和窗口标题。
- macOS：osascript 获取前台应用和窗口标题。
- Windows Chrome/Edge：尝试用 UIAutomation 读取地址栏 URL。
- macOS Chrome/Edge/Safari：尝试读取当前标签 URL。
- 获取 URL 成功时按域名分类；失败时降级到标题关键词。
- 可推断上下文：
  - browser
  - editor
  - ai
  - terminal
  - dev-server
  - generic

注意：浏览器应用会优先标记为 `browser` 上下文；AI 网页是否放行主要由 URL/域名/标题关键词规则决定。

默认放行的 AI/工作上下文包括：

- Kimi / Moonshot
- Claude
- ChatGPT / OpenAI
- 豆包 / Doubao
- GitHub
- Stack Overflow
- MDN
- localhost / 127.0.0.1

### 2.7 统计与 telemetry

已实现：

- 今日专注分钟。
- 今日完成任务数。
- 连续天数。
- 星尘。
- 当前专注评分。
- 漂移秒数。
- 黑名单秒数。
- 中立应用秒数。
- 允许应用秒数。
- 风险切换数。
- 工作流切换数。
- 总切换数。
- 拉回次数。
- 恢复次数。
- 最长漂移。
- 最近专注记录。
- 今日 / 7 天 rollup：
  - 专注分钟
  - 拉回次数
  - 平均评分

## 3. 当前未完成能力

### 3.1 高优先级

1. 更明显但不占位的面板唤醒入口  
   当前主要依赖右键/双击。下一步建议增加 hover 后出现的极小 `⋯` 按钮。

2. 浏览器 URL 识别增强  
   当前 URL 获取依赖平台权限和浏览器 UI 结构，可能失败。需要继续优化 Chrome/Edge/Safari 的可靠性。

3. 当前任务上下文映射  
   需要让任务知道“期望使用哪些应用/域名”，例如：
   - 写代码：VSCode / GitHub / docs / localhost / AI
   - 写作：文档工具 / AI / Notion
   - 学习：PDF / 课程网站 / 笔记软件

4. VSCode / Claude / 豆包 / 本地进程状态集成  
   当前只做轻量推断，还没有读取 VSCode workspace、终端任务、dev server 状态。

5. AI 拆解接 LLM  
   当前 `✨` 是模板拆解。下一步接入 LLM API，根据任务文本生成更贴合的子任务。

6. 任务上下文继续增强  
   当前已有任务上下文推断雏形，下一步需要让用户更方便地编辑 expected app/domain/keyword。

### 3.2 中优先级

- 每日/每周趋势图。
- 最容易分心应用/域名排行。
- 最佳专注时间段。
- 任务计划时间 vs 实际完成对比。
- CSV 导出。
- 托盘重置桌宠位置。
- 白名单/黑名单一键从当前应用添加。
- 主进程或 wall-clock 计时校正，减少渲染进程被系统挂起时的误差。

### 3.3 暂缓

用户已明确表示暂时不做：

- 养成系统。
- 多桌宠资产。
- 装扮商店。

这些保留为远期方向，不进入当前迭代主线。

## 6. 产品原则

- 桌宠是注意力锚点，不是骚扰源。
- 提醒要轻，不要羞辱。
- 专注模式优先“拉回”，不是“惩罚”。
- 配置要傻瓜化，不能让用户为了专注而先做一堆设置。
- 对 ADHD 用户，启动比完美更重要。
